# RAG — PDF Question Answering

A Retrieval-Augmented Generation (RAG) application that lets you upload PDF documents and ask natural language questions about their content. Answers are generated by a Groq-hosted LLM and grounded entirely in your uploaded documents.

## Features

- **PDF Upload & Processing** — Upload one or more PDFs; text is automatically extracted, chunked, and indexed.
- **Semantic Search** — Questions are embedded and matched against document chunks using cosine similarity.
- **AI-Powered Answers** — Groq's LLaMA 3.1 model generates concise answers based only on retrieved context.
- **Chat Interface** — Streamlit-based conversational UI with message history and source attribution.
- **Persistent Vector Store** — ChromaDB stores embeddings on disk so your knowledge base survives restarts.

## Tech Stack

| Component        | Technology                          |
|------------------|-------------------------------------|
| UI               | Streamlit                           |
| PDF Parsing      | pypdf                               |
| Text Chunking    | LangChain Text Splitters            |
| Embeddings       | Sentence Transformers (MiniLM-L6)   |
| Vector Database  | ChromaDB                            |
| LLM              | Groq (LLaMA 3.1 8B Instant)        |

## Setup

### 1. Clone the repository

```bash
git clone <repo-url>
cd "RAG Project"
```

### 2. Install dependencies

```bash
pip install -r requirements.txt
```

### 3. Configure API key

Create a `.env` file in the project root:

```
API_KEY=your_groq_api_key_here
```

Get a free API key from [console.groq.com](https://console.groq.com/).

### 4. Run the application

```bash
streamlit run app.py
```

## Usage

1. **Upload** — Use the sidebar to upload one or more PDF files.
2. **Process** — Click "Process Documents" to extract, chunk, embed, and store the content.
3. **Ask** — Type a question in the chat input at the bottom of the page.
4. **Review** — Expand the "Retrieved Context" section to see which document chunks informed the answer.

## Configuration

All configurable values live in `src/config.py`:

| Parameter         | Default              | Description                              |
|-------------------|----------------------|------------------------------------------|
| `EMBEDDING_MODEL` | `all-MiniLM-L6-v2`  | Sentence Transformers model name         |
| `LLM_MODEL`       | `llama-3.1-8b-instant` | Groq LLM model                       |
| `CHUNK_SIZE`      | `500`                | Maximum characters per chunk             |
| `CHUNK_OVERLAP`   | `100`                | Overlapping characters between chunks    |
| `TOP_K`           | `3`                  | Number of chunks retrieved per query     |
